Q1: Расскажите, чем отличается хороший текст от плохого. Приведите примеры.

Хороший текст решает задачу. Чем точнее и быстрее решена задача, тем лучше текст. Худший текст не решает задачу. Решим задачу “прокатиться на лифте”.

Плохо:
  • Для вызова кабины лифта нажмите кнопку вызывного аппарата, установленного на этажной площадке.
  • При входе в кабину, убедитесь, что она находится перед вами.
  • Войдя в кабину, нажмите кнопку нужного Вам этажа.

    • Нормально:
  • Вызвать лифт — кнопка справа.
  • Не прыгайте в шахту, ждите лифт.
  • Выберите этаж, наслаждайтесь поездкой.

    • Хорошо: Текст не нужен. Все и так знают, как пользоваться лифтом.

    Q3: Расскажите, чем API отличается от IP, а DNS — от SMS.

    IP — правила объединения компьютеров в сеть. Благодаря IP компьютер 1 знает, как найти компьютер 2.

    API — правила совместной работы программ. Компьютерные программы используют API для обмена информацией друг с другом. Когда две программы передают друг другу информацию — это API. Если программы передают информацию через сеть, то к API добавляем IP.

    Чтобы избежать путаницы при обмене информацией, каждому компьютеру присваивают уникальный IP-адрес. Пример IP-адреса: 193.232.168.26. Человеку трудно запомнить такие адреса. Поэтому придумали специальную систему — DNS. DNS переводит адрес из цифр в адрес из букв: IP-адрес 193.232.168.26 превращается в funbox.ru. Адрес из букв понятнее и проще запомнить.

    Не путайте DNS и SMS. SMS — текстовые сообщения, которые приходят на телефон.

    Level 2.

    Task 1. Отредактируйте начало инструкции по работе с API Blueprint.

    Введение

    API Blueprint или APIB — язык документации API. APIB описывает взаимодействие клиента с сервером: доступные запросы, параметры URL, формат данных. APIB использует разметку Markdown. Данные внутри документации описывают с помощью JSON и MSON.

    Изучите спецификацию APIB и МSON. Там много нюансов.

    Инструменты

    Создать документацию в формате APIB можно в текстовом редакторе: VS Code, Atom, Vim.

    Для совместной работы используйте рендер Markdown. Рендер создаст HTML-страницу из файла APIB. Так удобнее показывать документацию.
    Преобразуйте текст документации в AST с помощью парсера.
    Для фронтенда используйте валидатор ответов сервера. Для WebStorm установите плагин автодополнения URL.
    Для бэкенда используйте плагин проверки ответов контроллеров.


    Task 2. Отредактируйте часть сценария работы чат-бота.

    Отправка и прием сообщений 1-1 чат-ботом

    Определения:

    КЛИЕНТ — человек, пользуется мобильной связью.
    БОТ — программа. БОТ переписывается с клиентом.
    БОТ выполняет функцию посредника между сервером и КЛИЕНТОМ

    Шаг 1. БОТ проверяет связь с КЛИЕНТОМ.

    БОТ отправляет запрос OPTIONS. Запрос OPTIONS узнаёт функциональные возможности КЛИЕНТА. КЛИЕНТ отвечает на запрос OPTIONS.
    Если заголовок Contact содержит 200OK — ответ положительный, связь с КЛИЕНТОМ установлена. БОТ продолжает сценарий.
    Если заголовок Contact не содержит 200OK — ответ отрицательный, нет связи с КЛИЕНТОМ. Логика отправки сообщений получает негативный ответ. Прикладная логика выбирает другой канал доставки сообщения.

    Шаг 2. БОТ создаёт чат 1-1 с КЛИЕНТОМ

    БОТ запрашивает сервер создать новый чат.    Сервер создаёт чат 1-1. БОТ отправляет КЛИЕНТУ запрос INVITE. Запрос INVITE добавляет КЛИЕНТА в чат 1-1.    
    Запрос INVITE уточняет заголовки. Заголовки без уточнений заполнять по шагу 1.


    Task 3. Отредактируйте начало статьи о типографике в вебе.

    Типографика — искусство оформления текста, базовый навык дизайнера и верстальщика. Этот документ рассказывает принципы типографики. Материал будет полезен, если вы создаёте сайты и приложения.

    Правила типографики легко запомнить. Сложно научиться применять знания на практике. Похоже на русский язык — каждый изучал в школе, но не каждый умеет говорить и писать без ошибок. Профессионала и любителя отличают по оформлению текста. Хорошая типографика означает строгое соблюдение правил.
